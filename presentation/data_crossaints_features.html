03 — AI Assistant

Conversational AI Analytics

Natural language queries with structured, data-grounded responses

Three Response Modes

Done
  • Clarify: Smart follow-up questions with suggested options when query is vague
  • Configure: Proposes dashboard filter/view changes applicable with one click
  • Analyse: Structured investigation with planning, querying, and synthesis

Facts / Interpretation / Hypothesis

Done
  • Every response structured into XML-tagged sections with clear epistemological boundaries
  • Facts (data points) → Interpretation (business context) → Hypothesis (root cause) → Recommendations (actions)
  • AI investigates 2-4 angles per question using 10+ data query tools before synthesising

Context & Interaction

Done
  • Receives current page, filters, period, comparator, and scale
  • Right-click any data point (long-press on mobile) to ask AI about it
  • Voice input via Web Speech API, multi-chat sessions (up to 20), SSE streaming

Unified AI Engine

Done

One investigation pattern shared between assistant chat and automated insights. A semantic model dynamically built from actual data at startup ensures the AI always references real entities. Both modes share the same query tools and output structure — chat adds dashboard context and render tools; insights add anomaly details, severity classification, and positive/negative outcome tagging.

04 — Roles & Access

Role-Based Personalisation

Different perspectives for different users — same trusted data foundation

What Works Today

Done
  • Each role defines: prompt personality, available AI tools, and insight visibility scope
  • The AI adapts its language, framing, and focus to match the user's role
  • Tool access is filtered — e.g. Market Lead cannot query expense or org-unit data
  • Insight scope filtering — users only see anomalies relevant to their domain

Four Defined Roles

Done
  • Enterprise Analyst (default): Full access to all tools, insights, and dimensions
  • CFO: Focus on margin, OpEx efficiency, forecast accuracy. Framed through P&L impact
  • Commercial Lead: Brand performance, market share, competitive positioning. No expense data
  • Market Lead: Scoped to specific market data and insights only

What's Still Needed

Partial
  • Data route enforcement: Roles filter AI tools and insights, but dashboard API endpoints are not yet restricted by role — any user can query any data view directly
  • Default page & settings: Role configs declare default_page, but the frontend doesn't read it yet — all users start on Overview
  • Frontend role context: No role selector or role-aware UI — role is only applied server-side via X-User-Id header for insights

Detection Methods

Done
  • Outlier (Trend-Residual): Fits a linear trend on history, flags points that deviate significantly from the projected trend — both above and below. Eliminates false positives from steady growth
  • Drift (Slope Change): Compares the slope in the recent window vs historical window. Only fires on genuine acceleration or deceleration, not steady trends. Scored as % of series mean
  • Target Miss: Actual vs budget variance beyond configurable threshold (default 15% at low sensitivity)
  • Competitive Shift: Period-over-period market share changes beyond threshold

Frequency-Adaptive & AI-Classified

Done
  • Auto-detects data frequency (monthly, weekly, daily) and adjusts window sizes — no config needed
  • AI classifies each anomaly as positive (revenue beat, share gain) or negative (shortfall, cost overrun)
  • Three sensitivity profiles (high/medium/low) configurable via env var or JSON
  • Atomic file store with cross-platform locking (Windows & Unix)

Push vs Pull

Done
  • Push: Web Push for critical anomalies, filtered by role scope
  • Pull: All insights available in the insights panel on demand
  • Alert fatigue mitigated by dedup, severity gating, and AI push-eligibility

07 — Gaps & Roadmap

What's Not Yet Built

Remaining hackathon requirements and future capabilities

Not Started

  • External signals: No integration with competitor data, news, or AZ public filings — all analysis uses internal data only

Partially Built

  • Role enforcement: AI personalisation and insight scoping work, but data APIs are unrestricted and frontend has no role awareness
  • Saved views: Dimension configs can be saved; full dashboard state bookmarks not yet wired to UI
  • Scaling beyond Finance/Commercial: Architecture is domain-agnostic but only revenue, expenses, and market share are loaded
  • Production readiness: No auth middleware, no persistent database, JSON file storage

Easy to Add

  • External data: Add a web search tool to the AI tool system
  • Role enforcement: Add middleware to check role on data routes; wire default_page to frontend
  • Saved views: useBookmarks hook exists; needs UI button
  • New data domains: Just add CSV files and dimension config
