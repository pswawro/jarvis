Requirement 2 of 5 · Standard vs. Personal

Governance Meets Flexibility

"What must remain standardised? Where does personalisation add real value?"

Standardised (locked)

  • 3 hierarchy views: Brand→TA→Total, Market→Region→Total, Unit→Total
  • Consistent columns: Actual, Budget, Variance%, Sparkline across all views
  • Single data source: All views and AI query same API endpoints
  • Reconciled totals: Root node always shows company-level aggregates
  • Period standardisation: YTD/Monthly/Quarterly via shared toggle

Personalised (flexible)

  • Bookmarks: Save & restore favorite filter/view combinations
  • Comparator choice: Budget / Prior Year / Forecast / MTP / RBU2
  • AI chat context: Assistant adapts to current dashboard state & filters
  • Drill-down freedom: Expand any node to explore hierarchy depth
  • Hub navigation: Role-specific entry points (CFO view, commercial view)
💬

Discussion: RBAC is required by the brief ("CEO sees everything, market user sees only their market"). We haven't implemented access control yet. For the demo, should we mock it with a user-switcher dropdown, or explain it as a roadmap item? Also — bookmarks currently save to localStorage. Production would need server-side persistence.

Requirement 3 of 5 · Root Cause Clarity

Beyond Surface-Level Variance

"How do we help users uncover the underlying business reason behind a variance?"

What Works Today

  • AI queries multiple data levels to build explanations (TA → brand → market)
  • Uses think tool for multi-step reasoning before answering
  • Cites specific brands, markets, and dollar amounts from real data
  • Cross-references financial outcomes with commercial performance
  • Tested: 3/3 runs consistently identify same top underperformers

Waterfall Decomposition

  • decompose_variance tool — AI builds structured breakdowns
  • Visual waterfall/bridge chart: green bars (positive) + red bars (negative) = total
  • Each factor = specific entity (brand, market, unit) with $ contribution
  • Connects "what happened" to "why" — e.g., TAGRISSO US -$120M due to patent cliff
  • Renders as clickable overlay in chat, same as tables/charts

Example: "Why is Oncology underperforming?"

AI correctly identifies Imfinzi, Enhertu, and Truqap as key drivers across all 3 test runs. Cross-references market share data with revenue variance. Distinguishes between volume decline and pricing pressure.

💬

Discussion: Both waterfall decomposition and actionable recommendations are implemented. Should we highlight these in the demo flow as separate moments, or let them emerge naturally from questions? The recommendations section is a direct answer to the brief's "recommendation of actions" requirement.

Requirement 4 of 5 · AI Augmentation

AI That Augments, Not Replaces

"Maintaining clear boundaries between facts, interpretations, and hypotheses"

Facts

Data-grounded statements with specific numbers from API queries. Always sourced from real data tools.

"Total Revenue for 2025 YTD is $50,195.8M, +2.3% vs budget."

Interpretation

AI analysis connecting data points. Explains patterns and relationships the user should know about.

"CVRM is driving the positive variance, compensating for Oncology's miss."

Hypothesis

Forward-looking suggestions or theories that go beyond the data. Explicitly marked as uncertain.

"If Enhertu's Q3 pipeline issues persist, FY could miss by ~$200M."

AI Architecture

Tool-Grounded Responses

  • 15 LLM tools: query_kpi, query_brand, query_region, query_unit, query_market + chart variants
  • Fabrication guard: Detects and blocks hallucinated tables
  • Scope awareness: Distinguishes "top 5 overall" from "top 5 in current TA filter"
  • Clarification mode: Asks follow-up for vague queries instead of guessing

Transparency Features

  • Thinking steps: Shows AI reasoning chain (which tools called, why)
  • XML tag separation: <facts>, <interpretation>, <hypothesis>, <recommendations>
  • Visual overlays: Tables and charts rendered with source attribution
  • Config proposals: AI suggests dashboard changes, user confirms

Testing · AI Quality

AI Consistency & Accuracy

Bugs found, fixed, and validated through systematic testing

Bug Found: Filter Leak

Asking "top 5 brands overall" while Oncology filter was active — AI passed TA filter to query, returning only Oncology brands instead of cross-company ranking.

Fix: Added scope awareness to system prompt — AI now distinguishes scoped vs. cross-scope queries. Verified across 3 runs.

Bug Found: False Error Message

Fabrication guard triggered "Error: I wasn't able to process that" even when the AI had correctly used render_table tool — because guard didn't know tools had been called in prior iterations.

Fix: Added used_tools flag to skip guard when tools were previously invoked. Zero false positives since.

Model Configuration (Implemented)

  • Temperature: 0.1 (low for analytical accuracy)
  • Default model: Claude Sonnet 4.6 via LLM_MODEL_ID env var
  • Heavy model: Claude Opus 4.6 via LLM_MODEL_ID_HEAVY env var
  • Auto-routing: Iterations 0-1 use Sonnet, 2+ use heavy model
  • Debug mode: LLM_DEBUG=true logs all tool parameters

Test Infrastructure

  • API-level tests: pytest suite hits /api/assistant directly
  • Ground truth validation: Cross-checks LLM numbers against data API
  • Multi-run consistency: Each test runs 3x to catch non-determinism
  • Model comparison: compare_runs.py diffs Sonnet vs. Opus results
  • Results saved as JSON for cross-run analysis
💬

Discussion: Should we run the full test suite with Opus and compare accuracy/cost? Current tests pass with Sonnet, but Opus may handle edge cases better. Cost estimate: ~$2-3 per full test run with Opus vs. ~$0.30 with Sonnet.
