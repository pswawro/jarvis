03 — AI Assistant

Conversational AI Analytics

Natural language queries with structured, data-grounded responses

Three Response Modes

Done
  • Clarify: When user query is vague, AI asks smart follow-up questions with suggested options
  • Configure: When user wants to navigate ("show me Oncology in US"), AI proposes dashboard filter/view changes that can be applied with one click
  • Analyse: For data questions, AI runs a structured investigation with planning, querying, and synthesis

Structured Investigation

Done
  • AI makes its reasoning visible via think steps (plan → finding → pivot → synthesis)
  • Queries real data using 10 internal tools (KPI, brand, region, unit, market, charts, config)
  • Multi-turn: investigates 2-4 angles before synthesising
  • Renders inline tables, charts, and variance decomposition waterfalls

Facts / Interpretation / Hypothesis

Done
  • Every analytical response is structured into clear XML-tagged sections
  • Facts: Data points discovered, with numbers
  • Interpretation: What the data means in business context
  • Hypothesis: Likely root cause or explanation
  • Recommendations: Specific, actionable next steps with named entities

Context Awareness

Done
  • Receives current page, filters, period, comparator, and scale
  • Right-click any data point to ask AI about it
  • Long-press on mobile to trigger AI investigation
  • Voice input via Web Speech API — tap microphone, speak, auto-submits transcription
  • Multi-chat sessions (up to 20) with localStorage persistence
  • Real-time SSE streaming of response sections
04 — Roles & Access

Role-Based Personalisation

Different perspectives for different users — same trusted data foundation

How Roles Work

Done
  • Each role defines: prompt personality, default page, available tools, and insight visibility scope
  • The AI adapts its language, framing, and focus to match the user's role
  • Tool access is filtered — a Commercial Lead cannot query expense data
  • Insight scope filtering — users only see anomalies relevant to their domain

Defined Roles

Done
  • Enterprise Analyst (default): Full access to all tools, insights, and dimensions
  • CFO: Focus on margin, OpEx efficiency, forecast accuracy. Framed through P&L impact
  • Commercial Lead: Brand performance, market share, competitive positioning. No expense data
  • Market Lead: Scoped to specific market data and insights

Hackathon Alignment: Access Security

The brief requires "grant different access to data to different people (e.g. CEO can see everything while somebody in a market can only see data from such market)." Jarvis implements this through role-based configuration: each role defines which data tools are available and which insight scopes are visible. The semantic model is injected into every prompt, ensuring the AI can only reference data it has access to query.

Done
05 — Push Insights

Automated Anomaly Detection Pipeline

From statistical detection to AI analysis to push notification

1

→
2

Each anomaly gets a stable identity for deduplication and escalation tracking
→
3
AI Analyse
Claude investigates context, queries data tools, and writes structured analysis
→
4
Classify
AI assigns severity (critical / notable / informational) and push eligibility
→
5
Deliver
Web Push notification to subscribed users, filtered by role scope

Four Detection Methods

Done
  • Outlier (Z-Score): Flags single-month values that deviate significantly from historical average
  • Drift (Rolling Mean): Detects sustained trend changes — recent window vs overall baseline
  • Target Miss: Compares actual revenue vs budget targets, flags when variance exceeds threshold
  • Competitive Shift: Detects month-over-month market share changes beyond threshold

Entities Monitored

Done
  • Revenue: Every brand-market combination, TA aggregates, and total company
  • Expenses: Sub-unit level, unit aggregates, and total company
  • Competitive: Brand-market pairs with competitive category tracking
  • Configurable sensitivity profiles (zscore thresholds, rolling window, market share delta)
05 — Push Insights (cont.)

Insight Management & Interaction

Avoiding alert fatigue through smart deduplication and user controls

Smart Deduplication & Escalation

Done
  • Fingerprinting prevents the same anomaly from creating duplicate alerts across runs
  • If an existing anomaly worsens (>30% score increase), the insight is automatically re-analysed by AI
  • When an anomaly is no longer detected, its insight transitions to "inactive" status
  • Push notifications only fire for genuinely new or escalated findings

User Interaction

Done
  • Insights panel: Slide-out panel with severity-sorted list of anomalies
  • Read/unread tracking: Unread count badge with blue dot indicators
  • Bookmarking: Save important insights for later review, filter to show bookmarks only
  • Send to chat: One-click to open AI assistant with full insight context for deeper analysis
  • Expandable analysis: Preview first section, expand to see full facts/interpretation/hypothesis/recommendations
  • Sort: By date (newest first) or by severity level

Hackathon Alignment: Push vs Pull

The brief asks: "Should insights be pushed or pulled? How do we avoid noise and alert fatigue?" Jarvis implements both: critical anomalies are pushed via Web Push notifications (filtered by role scope), while all insights are available to pull from the insights panel. Alert fatigue is mitigated by fingerprint-based deduplication, severity gating (only critical triggers push), and AI-driven push eligibility (the model decides if a finding truly warrants interruption).

Done
06 — AI Architecture

Unified AI Analysis Engine

One investigation pattern shared between assistant chat and automated insights

Shared Prompt Architecture

Done
  • Common core: personality rules, semantic model (brands, markets, org structure), and formatting conventions
  • Shared investigation pattern: plan → query → finding → drill/pivot → synthesise
  • Same XML output sections: facts, interpretation, hypothesis, recommendations
  • Same data query tools: 10 endpoints covering KPIs, revenue, expenses, and competitive data

Mode-Specific Extensions

Done
  • Assistant chat adds: dashboard state context, clarify/configure modes, render tools (tables, charts, variance decomposition), filter scope guidance
  • Insight analysis adds: anomaly details (detection type, entity, raw statistics), severity classification, push eligibility determination
  • No duplicated prompt content — each mode composes only the pieces it needs

Semantic Model

Done

Dynamically built from actual data dimensions at startup. Includes the complete brand portfolio grouped by therapeutic area, all markets with regional grouping, full organisational structure with units and sub-units, and competitive categories. This ensures the AI always references real entities and never fabricates brand or market names.

09 — Gaps

What's Not Yet Built

Remaining hackathon requirements and future capabilities

Not Started

  • External signals: "Integration of external signals from AZ and from competitors" — currently all analysis is based on internal data only

Partially Built

  • Saved views: Dimension configurations can be saved/loaded. A bookmark hook for full dashboard state (filters + view + period) exists but is not connected to the UI
  • Scaling beyond Finance/Commercial: Architecture supports new data domains, but only revenue, expenses, and market share are loaded. HR, manufacturing, and R&D mentioned in brief are not yet included
  • Internal context leverage: The semantic model provides structural context, but there is no integration with internal knowledge bases or prior analyses
  • Production deployment: Designed for Domino Data Lab but not production-hardened (no auth middleware, no persistent database, JSON file storage)

Easy to Add

  • External data: The AI tool system is extensible — adding a web search or news query tool would give the assistant external context
  • Saved views: The useBookmarks hook is already built; it just needs to be wired to a UI button
  • New data domains: Data loader and tree builder are domain-agnostic; adding HR/R&D data requires only new CSV files and dimension config
